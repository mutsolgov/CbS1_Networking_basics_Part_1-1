#### Подсети для организации N
У нас есть 3 сегмента:
1. **Технологический сегмент** – 3 устройства  
2. **Серверный сегмент** – 16 устройств  
3. **Пользовательский сегмент** – 32 устройства  

Каждый сегмент требует определенного количества IP-адресов, включая один адрес сети и один широковещательный адрес.

- **Технологический сегмент (3 устройства)**  
  Минимальный размер сети = 3 + 2 (адрес сети + broadcast) = **5 адресов**  
  Ближайшая подходящая маска: `/29` (255.255.255.248) – даёт 8 адресов (2^3)

- **Серверный сегмент (16 устройств)**  
  Минимальный размер сети = 16 + 2 = **18 адресов**  
  Ближайшая подходящая маска: `/27` (255.255.255.224) – даёт 32 адреса (2^5)

- **Пользовательский сегмент (32 устройства)**  
  Минимальный размер сети = 32 + 2 = **34 адреса**  
  Ближайшая подходящая маска: `/26` (255.255.255.192) – даёт 64 адреса (2^6)

Таким образом, маски для сегментов:  
- **Технологический сегмент:** `/29` (255.255.255.248)`  
- **Серверный сегмент:** `/27` (255.255.255.224)`  
- **Пользовательский сегмент:** `/26` (255.255.255.192)`
